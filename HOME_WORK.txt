Описание занятия: 
+1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
+2.Как вернуть файл из индекса в рабочий каталог?
+3.Как отменить ошибочный commit ?
+4.Как создать новую ветку и переключится в нее?
+5.В чем отличение между fast-forward merge  and non fast-forfard merge?
+6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
+7.Можно ли удалить ветку? Как? 


В ответах на вопросы 1-4, 7 необходимо указать конкретные команда git. 
На 5 и 6 вопросы дать объяснение своими словами.

Ответы:
1. git checkout <file name>(+ "." отмена во всех файлах)
2. git restore --staged <file name> (git reset)
3. git reset HEAD~(кол-во шагов назад, локально) (git commit --amend -m "<commit>" //отредактировать последние commit)
   git revert HEAD (откатить и отменить последеий commot и сделать зеркальный commit) - самый правильный способ
4. git branch <name>// создаст новую ветку но мы остаёмся в <mane
   git checkout -b <name>// создаст новую ветку и сразу перейдёт в неё
5. fast-forward merge используем когда текущая ветка не имеет новых коммитов а ветка, которую мы мержим имеет.
   non fast-forfard merge используется при мерже когда для текущей ветки автоматом создаётся новый коммит, чтобы сохранить старые коммиты и к ним прикрепить новый коммит от вливаемой ветки
6. Может возникнуть конфликт слияния веток когда вносились изменения в один и тот же файл в разных ветках до слияния. Разрешить это конфликт можно вручную отредактировав файл (fix conflicts and then commit the result.)
7. Ветку можно удалить командой  git branch -d
